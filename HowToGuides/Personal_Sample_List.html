Set Up

The goal of this tutorial is to demonstrate how to parse through the TRGAted application data based on your personal groupings. This tutorial will principally cover the basics of a categorical survival comparison based on your groupings and a continuous survival comparison based on your groupings. The tutorial is concerned with parsing through the sample information in one of the cancer types available through the TRGAted app.

Processing the samplelist

TRGAted application runs on Patient ID, we need to filter for sample IDs containing -01 (primary tumor) and then remove the -01 from the ID to convert to Patient ID.

samplelist <- dplyr::filter(samplelist, grepl("-01",ID)) #For Sample IDs to remove -01
samplelist$ID <- substr(samplelist$ID,1,nchar(as.character(samplelist$ID))-3) #For Sample IDs to remove characters after the hyphen to convert them to Patient ID
samplelist<- samplelist[order(samplelist$ID),] #Alphabetically orders data frame by Patient ID

Load Cancer Data

This could be any .csv file located in the TRGAted github repository.

cancerType <- read.csv("~/TRGAted/BRCA.csv")
cancerType <- cancerType[cancerType$patient %in% samplelist$ID,] #list removes samples not in the samplelist
cancerType <- merge(samplelist, cancerType, by.x="ID", by.y = "patient") #merges your list with the TRGAted App Data

Analysis

Survival analysis based on categorical groupings

Here we are going to examine the difference in overall survival outcomes based on the HER2 versus basal (or basal-like) subtypes of breast cancer.

Status = "OS" #modify this to change to DSS, DFI or PFI
Time = "OS.time" #modify this to change to DSS.time, DFI.time, or PFI.time


fit <- survfit(Surv(OS.time, OS) ~ group,
               data = cancerType) #This generates the survival fit for curve generation

ggsurvplot(fit, data = cancerType,
                tables.y.text = FALSE,
                tables.theme = theme_cleantable(),
                xlab = "Time in days",
                pval = T,
                pval.coord = c(0.05, 0.05),
                legend.labs = c("basal", "HER2"),
                risk.table = TRUE, 
                fontsize = 5,
                ggtheme = theme_bw(base_family = "sans", base_size = 12))

Hazard Ratio

To get the hazard ratio, we use the cox proportional hazard function with the same equation as above. The exp(coef), is the hazard ratio, in this instances HER2+ breast cancer is more hazardous in terms of overall survival (this is already known), with a p-value approaching significance at 0.1.

coxph(Surv(OS.time, OS) ~ group,
               data = cancerType)

Survival analysis based on categorical groupings and the continous RPPA values

Here we are going to expand our survival comparison between HER2 and basal breast cancers, by adding the RPPA quantification for the estrogen receptor (ERALPHA).

cut <- surv_cutpoint(cancerType, Time, Status, variables=c("ERALPHA"), minprop = 0.15) #Optimal Cutpoint for survival

cat <- surv_categorize(cut)
cat <- cbind.data.frame(cat, cancerType$group)
colnames(cat)[4] <- "group"

fit2 <- survfit(Surv(OS.time, OS) ~group + ERALPHA,
               data = cat)

ggsurvplot(fit2, data = cancerType,
                tables.y.text = FALSE,
                tables.theme = theme_cleantable(),
                xlab = "Time in days",
                pval = T,
                pval.coord = c(0.05, 0.05),
                risk.table = TRUE, 
                legend.labs = c("basal-ERlow", "basal-ERhi", "HER2-ERlow", "HER2-ERhi"),
                fontsize = 5,
                ggtheme = theme_bw(base_family = "sans", base_size = 12))

Hazard Ratio

The hazard ratio for this comparison can be performed similarly, but will only compare the two variables in our occasion. Here you see that ERALPHA-low levels are associated with a worse overall survival (HR=0.264, p=0.0013) and similar to above HER2 has worse outcomes, but does not reach the significance threshold.

coxph(Surv(OS.time, OS) ~ group + ERALPHA,
               data = cat)
